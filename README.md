# Групповой проект №2. DE: Scraping & API
## Описание предметной области
В рамках данного группового проекта нами будут использованы инструменты парсинга и скраппинга для сбора и анализа данных. Нами была выбрана такая предметная область, как вселенная видеоигр :)
## Описание этапа парсинга
В первую очередь, мы определились, что парсить будем информацию из idgb - публичной базы данных о различных видеоиграх, которая принадлежит Twitch. Для начала мы собрали "вспомогательную информацию", такую как информацию о жанрах игр, платформах, режимах игр, тематиках (отличается от жанров тем, что жанр - то, каким образом происходит игра (рпг, шутер, рогалик и так далее), а тематика - общее состояние вокруг, атмосфера (хоррор, фантастика и так далее)), статусах игр (ранний доступ, релизнуто, отменен релиз и тому подобное), а также о типе игры как таковой (то есть это может быть основная игра, dlc, какой-то аддон и тд). Также была вытащена информация о возрастных категориях (как и в ГП1, там все разделялось на возрастные категории и их различные обозначения от разных организаций, которые выставляли конкретное возрастное ограничение игре). Ну и, конечно же, был собран датасет игр (при этом мы учитывали только те игры, у которых было более 5 оценок, чтобы +- ноунейм игры не попадали). Этот датасет далее будет мерджиться с датасетом, который собирается скраппингом (о нем чуть позже).
### Описание признаков
1. Справочник жанров genres
- id - айдишник жанра (тип integer)
- name - название жанра (тип string)
2. Справочник платформ platforms
- id - айдишник платформы (тип integer)
- name - название платформы (тип string)
- abbreviation - сокращение платформы (тип string)
В abbreviation могут быть пропуски, так как у некоторых платформ отсутствуют сокращения, либо же они не внесены в базу данных
3. Справочник режимов игр game_modes
- id - айдишник режима (тип integer)
- name - название режима (тип string)
4. Справочник тематик игр themes
- id - айдишник тематики (тип integer)
- name - название тематики (тип string)
5. Справочник статусов игр game_statuses
- id - айдишник статуса (тип integer)
- status - статус релиза (тип string)
- created_at - дата создания статуса (тип datetime)
- updated_at - дата обновления статуса (тип datetime)
6. Справочник типов игр game_types
- id - айдишник типа (тип integer)
- type - тип игры (тип string)
- created_at - дата создания типа (тип datetime)
- updated_at - дата обновления типа (тип datetime)
7. Информация о возрастных рейтингах age_ratings_merged
- id - айдишник возрастного рейтинга (тип integer)
- name - название рейтинговой организации (тип string)
- rating - возрастной рейтинг (тип string)
- description - расшифровка рейтинга по ключевым пунктам, которые характеризуют рейтинг (тип string)
8. Итоговая информация о признаках датасета, который является результатом работы парсингом games_complete
- id - айдишник игры (тип integer)
- first_release_date - дата первого релиза (тип datetime)
- game_modes - режимы игр (тип string)
- genres - жанры игр (тип string)
- name - название игры (тип string)
- platforms - платформы, на которых игра доступна (тип string)
- rating - рейтинг игры (тип float)
- summary - краткое описание игры (тип string)
- themes - тематики игры (тип string)
- total_rating - общий рейтинг игры (тип float)
- total_rating_count - количество оценок (тип integer)
- game_type - тип игры (тип integer)
- storyline - сюжетка (тип string)
- age_ratings - возрастной рейтинг (тип list)
- game_status - статус игры (тип float)
- aggregated_rating - рейтинг критиков (тип float)
- aggregated_rating_count - количество оценок критиков (тип integer)
- hypes - показатель интереса пользователей к игре (тип float)
Исходя из информации о количестве пропусков, можно сделать выводы, что в важных признаках пропуски - просто отсутствие информации в базе данных (в целом информацию можно найти, например, если пропуски в жанрах, платформах, рейтинге и тп), а отсутствие значений в специфичных полях (например, сюжет, тематика, хайа и тому подобные) - просто отсутствие информации, ибо она является не особо важной, например, не у всех игр есть хайп (то есть не за всеми играми следили люди, либо же следили, но явных доказательств этого нет). Признаки с большим количеством пропусков - в основном не несущие важной информации, которая влияет на целевую переменную - рейтинг
